object-fit: cover

Ensures image fits within the defined height and width attributes without stretching. One direction will always be bound, the other will be clipped and aligned to the center.

This example also demos a mobile adjustment class mentioned in the last example.

object-fit: cover + object-position: # #

Allows image to be nudged for preferred focal to be visible. Accepts two values (x and y distance) however when image is bound by one parameter*, the input value would be ignored. I.e. if container definition is a shorter aspect ratio than source image, the y value will adjust the vertical alignment, but an x value input will be ignored.

*When images and containers are set to fixed values and not %s, generally both values can used. Since most images we use are set to 100% width, generally only one parameter would have an impact.

Using flex-boxes instead of tables to make the box heights match

One struggle I noticed was the mis-matched box heights using the standard table method. This can be avoided using flex-boxes, and even more complex layouts can use an align property set to stretch to fill a space.

This example also uses a class defined as:

    @media screen and (max-width: 980px) {
        .mobile-full-width {
            width: 100% !important;
        } 
    }

If we could include a class like this somewhere in the program's css, it would allow us to more quickly adjust an element for mobile display. Whenever a width is defined as less than 100% (as the tables in the examples above were all set to 49% width), that width persists in mobile. If this class was inserted via class="mobile-full-width" in those same instances, the items would display at full device with when on small screens.
